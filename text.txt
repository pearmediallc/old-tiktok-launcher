Excellent — you’re now at the adgroup creation step in the TikTok Business API, and you want your ad schedule to follow Colombia time (UTC-05:00).

Let’s go through this carefully — because the TikTok API doesn’t directly let you set the timezone per adgroup, it uses the advertiser’s timezone.
But you can still fully control when your ad runs in Colombian time.

🧩 TikTok API rule

The adgroup creation endpoint:

POST /open_api/v1.3/adgroup/create/


or

POST /open_api/v1.3/adgroup/update/


Fields like start_time and end_time are Unix timestamps (in seconds) in UTC.

TikTok then interprets them according to the advertiser’s timezone.

✅ Step 1. Make sure the advertiser timezone = Colombia

You can’t directly set timezone in adgroup creation.

You can check your advertiser’s timezone with this endpoint:

GET https://business-api.tiktok.com/open_api/v1.3/advertiser/info/


Example response:

{
  "code": 0,
  "message": "OK",
  "data": {
    "advertiser_info": {
      "advertiser_id": "1234567890",
      "timezone": "America/Bogota",
      "timezone_offset": -5
    }
  }
}


If it shows "America/Bogota" or offset -5, you’re good.
If it shows "America/Mexico_City" but offset -5, that’s also fine — TikTok just labels it differently; both use UTC-05:00.

✅ Step 2. Convert your local Colombia time to UTC timestamps

Let’s say you want your ad to start at:

🕘 November 15 2025, 09:00 AM Colombia time (UTC-05:00)
🕔 November 15 2025, 17:00 PM Colombia time (UTC-05:00)

Convert both to UTC:

09:00 (UTC-5) → 14:00 UTC

17:00 (UTC-5) → 22:00 UTC

Unix timestamps (you can generate online or with code):

start_time = 1763205600
end_time   = 1763230800

✅ Step 3. Create adgroup with those timestamps

POST
https://business-api.tiktok.com/open_api/v1.3/adgroup/create/

Headers

Authorization: Bearer YOUR_ACCESS_TOKEN
Content-Type: application/json


Body

{
  "advertiser_id": "1234567890",
  "campaign_id": "1122334455",
  "adgroup_name": "Colombia Morning Campaign",
  "placement_type": "PLACEMENT_AUTO",
  "budget_mode": "BUDGET_MODE_DAY",
  "budget": 2000,
  "schedule_type": "SCHEDULE_FROM_CUSTOM",
  "start_time": 1763205600,
  "end_time": 1763230800,
  "promotion_type": "WEBSITE",
  "billing_event": "BILLINGEVENT_CLICK",
  "optimization_goal": "CLICK",
  "targeting": {
    "age": ["18-45"],
    "gender": "GENDER_ALL"
  }
}


✅ Result:
Even though the API interprets times in UTC, your ad will run exactly from 09:00 to 17:00 Colombia local time.
