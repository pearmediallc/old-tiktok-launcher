3. **Add/Update These Variables**


4. **Save Changes**
   - Render will automatically redeploy your app

## 🐛 Troubleshooting

### Error: 404 from TikTok

**Cause**: Redirect URI not registered in TikTok Developer Portal

**Solution**:
1. Register exact URL: `https://tiktok-launcher.onrender.com/oauth-callback.php`
2. Check for typos (must be exact match)
3. Wait a few minutes after registration

### Error: "Token exchange failed"

**Cause**: App credentials incorrect or redirect URI mismatch

**Solution**:
1. Verify `TIKTOK_APP_ID` and `TIKTOK_APP_SECRET` in Render
2. Ensure redirect URI matches exactly
3. Check app is approved in TikTok Developer Portal

### Error: Session/Cookie Issues

**Cause**: `SESSION_COOKIE_SECURE` not set for HTTPS

**Solution**:
1. Set `SESSION_COOKIE_SECURE=True` in Render environment variables
2. Ensure using HTTPS (not HTTP)

### Token Not Stored in localStorage

**Cause**: JavaScript blocked or browser privacy settings

**Solution**:
1. Check browser console for errors
2. Ensure localStorage is enabled (not in incognito mode)
3. Check browser doesn't block third-party storage

## 🔄 For Both Local and Production

If you want to support both environments, use this approach:

### Option 1: Two Separate .env Files

**.env.local** (for localhost):
```env
OAUTH_REDIRECT_URI=http://localhost:8080/oauth-callback.php
SESSION_COOKIE_SECURE=False
```

**.env.production** (for Render):
```env
OAUTH_REDIRECT_URI=https://tiktok-launcher.onrender.com/oauth-callback.php
SESSION_COOKIE_SECURE=True
```

### Option 2: Register Both URLs in TikTok

In TikTok Developer Portal, add **both** redirect URIs:
- `http://localhost:8080/oauth-callback.php`
- `https://tiktok-launcher.onrender.com/oauth-callback.php`

Then the `.env` file automatically switches based on where you run the app.
